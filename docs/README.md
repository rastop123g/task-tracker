# Task Tracker

## Краткое описание

Этот проект - трекер задач, основной целью которого является свести к минимуму количество необходимых операций в трекере задач, 
сохраняя при этом понятным процесс работы и возможности аналитики.

## Компоненты системы

- Фронтенд приложение на Vue.js
    - Приложение должно первостепенно отображаться в маленьком размере (как для мобильных устройств)
    - Только потом рассматривать масштабирование для больших экранов
- Бэкенд приложение rust
- Бекенд приложение на Node.js (под вопросом)
- База данных postgresql
- Кеш redis (позже)
- События nats (позже)
- Фронтенд (десктоп) приложение на Electron/Tauri (На базе приложения на Vue.js)

## Архитектурные принципы приложения

- Фронтенд приложение, хранит все данные, доступные пользователю локально в indexedDB
- Взаимодействие фронтенда с бекендом происходит через REST API а также через WebSockets (для рассылки событий по workspace)
- При запуске приложения Фронтенд приложение создает WebSocket соединение с бэкендом, а также запрашивает всю новую информацию,
за период off-line и сохраняет в IndexedDB
- Бэкенд приложение, в ws соединении хранит информацию о доступных workspaces для рассылки всех событий по нему
- Основная логика приложения (фильтации, сортировки, алертинги) выполняется на фронтенде на основе всех данных из IndexedDB и получаемых событий

## Пайплайн работы в приложении

1. Пользователь проходит процедуру регистрации/авторизации попадая на основную страницу приложения
    - Регистрация содержит Имя, email, password, аватарку
    - email необходимо подтвердить, только в этом случае будет работать авторизация
    - Авторизация производится по email и password
2. У нового пользователя доступно создание workspace или вход в существующий workspace по приглашению, а также создание presets для статусов
3. Пользователь создает workspace, создает статусы доступные в этом workspace, а также при желании пресеты создания задач, после чего может создавать задачи и приглашать участников по email
4. Приглашение работает следующим образом:
    1. Владелец workspace вводит email (сущ. пользователя) и нажимает пригласить
    2. У пользователя в приложении появляется уведомление о том что он был приглашен в workspace, а также, появляется алерт на кнопке настроек, и в пункте меню приглашения, где пользователь должен выбрать пункт войти в worspace
    3. Пользователь может выйти из workspace, при этом приглашение останется активным
    4. Владелец может как отозвать приглашение, так и исключить пользователя из workspace
5. Основная логика приложения
    - В верхнем меню приложения находится
        - Иконка настроек (где можно настроить свой профиль, пресеты статусов, выйти из аккаунта, выйти из workspace, создать workspace, зайти или пригласить, настроить уведомления по воркспейсам и статусам задач)
        - Название (аватарка) текущего workspace (кнопка переключения, со списком доступных workspaces и алертами на них)
            - хранится в localStorage
        - Фильтр по статусу задачи (поповер с чекбоксами)
            - хранится в localStorage и на бекенде
        - Фильтр по пользователю (только мои/все)
            - хранится в localStorage и на бекенде
        - Сортировка (по времени любого события - вверх/вниз)
            - хранится в localStorage
        - Поиск по задачам (по названию и описанию)
            - локальный поиск по задачам в списке задач
            - Позже имеет смысл сделать бекенд поиск по задачам, векторный включая сообщения в задаче
        - Кнопка создания задачи
    - клик в кнопку создания задачи показывает список presets для создания задачи а также перманентный элемент (задача с нуля)
    - ниже отображается список задач по воркспейсу с примененными фильтрами и сортировкой
        - элемент списка содержит 
            - ключ задачи
            - название (описание)
            - статус (если пользователь еще не видел этого статуса, это должно быть посвечено)
            - создателя (аватарка)
            - исполнителя(аватарку, но только если фильтр по пользователям - все))
                - при наведении тултип
            - Иконка (сообщения) с количеством новых сообщений в задаче которые еще не прочитаны
            - иконку продвижения по статусу
                - клик в иконку открывает поповер со всеми статусами задачи без ограничений и позволяет сменить статус у задачи
            - клик в задачу открывает поповер/модалку с деталями задачи
                - детали задачи содержат название (опционально), описание и чат по задаче (больше ничего лишнего)
                - опиcание задачи должно плавно перетекать в чат и с визуальной точки зрения быть частью чата
